# Excel 问卷转 SAV 开发方案

## 1. 目标

- 将现有问卷结果 Excel 文件转换为可在 SPSS 中直接使用的 `.sav` 文件。
- 保留问卷题目结构、单选题值标签、多选题二值结构、开放题文本。
- 为后续批量转换建立可复用的“映射表 + 转换脚本”机制。

## 2. 当前确认的前提

### 2.1 数据来源

- `data-value.xlsx`：主数据源，优先用于生成 `.sav`。
- `data-text.xlsx`：用于反推和校验值标签、变量标签、题目结构。

### 2.2 多选题存储方式

- 当前 Excel 文件中的多选题已经按“每个选项单独一列”展开。
- 在数值版中，多选题通常表现为：
  - `1` = 选中
  - `0` = 未选中
- 在文本版中，对应表现通常为：
  - `1` = 选中
  - 空白 = 未选中
- 因此，后续转换 `.sav` 时，按 **multiple dichotomy（多重二分法）** 处理，而不是按“一个单元格内多个编码”处理。

### 2.3 特殊字段

- `其他，请注明` 类型字段目前存在 `1KTV`、`1购买方便程度` 这类格式。
- 这类字段后续不建议直接原样进入分析版 `.sav`，建议拆分为：
  - 一个 `0/1` 二值变量
  - 一个文本补充变量

## 3. SAV 中变量、标签、值标签的处理方案

### 3.1 变量名（variable name）

- 不直接使用 Excel 原始列头。
- 使用短、稳定、ASCII 友好的命名方案，便于 SPSS 兼容和脚本处理。
- 命名示例：
  - 单选题：`q06_gender`
  - 单选品牌题：`q13_brand_main`
  - 多选题拆列：`q20_01`、`q20_02`
  - 开放题：`q23_text`
  - Van Westendorp 价格题：`q27a_low`、`q27b_value`、`q27c_high`、`q27d_too_high`
  - 元数据：`meta_submit_time`、`meta_duration`

### 3.2 变量标签（variable label）

- 使用完整中文题意，供 SPSS 中查看变量含义。
- 处理规则：
  - 单选题：完整题干
  - 多选题：完整题干 + 选项文本
  - 开放题：完整题干
- 示例：
  - `q06_gender` -> `3.记录用户性别`
  - `q20_01` -> `13.在选购啤酒时，除了品牌和价格之外，您最关注以下哪些因素？_原料/配料表干净`

### 3.3 值标签（value labels）

- 单选题：写入“编码 -> 文本”的映射关系。
- 多选题：统一写入：
  - `0` -> `未选`
  - `1` -> `选中`
- 文本题：不写值标签。
- 终止提醒列：
  - 默认不进入最终分析版 `.sav`
  - 如保留，则单独标记为流程变量，并视情况定义缺失值

### 3.4 多选题集合（Multiple Response Set）

- 基础版 `.sav` 可以先只写入多选拆列本身。
- 增强版 `.sav` 再定义 `MRSETS`，将同一题的多个 `0/1` 变量组成一个 multiple dichotomy set。
- `counted value` 统一使用 `1`。

## 4. 推荐的数据建模规则

### 4.1 单选题

- 每题对应一个变量。
- 变量中保留编码值。
- 文本含义通过值标签表达。

### 4.2 多选题

- 每个选项对应一个变量。
- 变量值为 `0/1`。
- 变量标签中保留选项文本。
- 同一题的多列后续可注册为一个 `MRSET`。

### 4.3 开放题

- 保留为字符串变量。
- 原始文本尽量不做破坏性清洗。

### 4.4 “其他，请注明”

- 拆成两列：
  - 主变量：`0/1`
  - 文本变量：去掉前缀 `1` 后的填写内容
- 示例：
  - `q38_99` = `1`
  - `q38_99_text` = `KTV`

### 4.5 元数据字段

- 提交时间、修改时间、录音地址、填表时长、地址、用户名称等字段暂作为辅助元数据保留。
- 在映射表中单独标记 `keep`，便于后续决定哪些字段进入最终分析版。

## 5. 映射文件设计

计划使用一个 Excel 映射模板管理转换逻辑，至少包含以下工作表。

### 5.1 `variables`

建议字段：

- `source_col`：原始 Excel 列号，如 `R`、`AV`
- `source_header`：原始列头
- `question_id`：如 `Q6`、`Q20`
- `spss_name`：SAV 变量名
- `variable_label`：SAV 变量标签
- `var_type`：`numeric` / `string`
- `measure`：`nominal` / `ordinal` / `scale`
- `role`：`single` / `multi_binary` / `open_text` / `meta`
- `keep`：是否输出到最终 `.sav`
- `missing_rule`：缺失值规则
- `notes`：备注

### 5.2 `value_labels`

建议字段：

- `spss_name`
- `code`
- `label`

### 5.3 `mrsets`

建议字段：

- `set_name`
- `set_label`
- `question_id`
- `set_type`：固定为 `multiple_dichotomy`
- `counted_value`：固定为 `1`
- `variables`：参与集合的变量名列表

## 6. 开发步骤

### 阶段 1：映射梳理

- 从现有分析结果中提取：
  - 单选题值标签
  - 多选题分组关系
  - `其他，请注明` 处理规则
- 生成首版映射模板。

### 阶段 2：转换脚本

- 编写 `src/` 下的转换脚本。
- 输入：
  - 原始 Excel 数据文件
  - 映射 Excel 模板
- 输出：
  - 标准化中间表
  - `.sav` 文件

### 阶段 3：增强元数据

- 为单选题写入值标签。
- 为多选题写入 `0/1` 值标签。
- 如工具链支持，则写入 `MRSETS`。

### 阶段 4：验证

- 用 SPSS 或兼容工具验证：
  - 变量名是否合规
  - 变量标签是否正常显示
  - 值标签是否正确
  - 多选题是否可正常分析

## 7. 工具选型建议

### 方案 A：基础稳定版

- 使用 `pyreadstat` 写出 `.sav`
- 优点：
  - Python 生态成熟
  - 足够支持变量标签、值标签、缺失值等核心信息
- 限制：
  - 对 `MRSETS` 支持有限，可能需要后处理

### 方案 B：增强完整版

- 使用 `pyspssio`
- 优点：
  - 更接近 SPSS 原生 I/O 能力
  - 更适合写入 multiple response sets
- 限制：
  - 环境要求更高
  - 接入和调试成本更高

### 当前建议

- 先做 `pyreadstat` 版，尽快产出可用 `.sav`
- 再根据需要评估是否补上 `MRSETS` 增强能力

## 8. 当前已知风险

- 原始列头过长，不能直接作为 SAV 变量名。
- `其他，请注明` 目前是混合格式，需要拆分。
- 少量开放题中出现类似编码串的内容，但不影响首版 `.sav` 生成。
- 某些题目的完整选项映射目前仅能基于样本中实际出现值确认，后续若拿到原始问卷题库，可进一步补全。

## 9. 下一步执行建议

按以下顺序推进：

1. 设计并生成映射 Excel 模板
2. 补全当前问卷的变量字典
3. 开发 Excel -> SAV 转换脚本
4. 产出首版 `.sav`
5. 验证并补强 `MRSETS`
